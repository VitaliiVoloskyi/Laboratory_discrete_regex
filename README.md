# Лабораторна робота: Regex FSM (Скінченний Автомат)
 
## Опис
 
Реалізація скінченного автомата (FSM) для компіляції та перевірки регулярних виразів із використанням патерну проєктування **State** (Стан). Автомат підтримує базовий функціонал regex: літери англійської абетки, цифри, оператор `.` (будь-який символ), а також квантифікатори `*` та `+`.
 
---
 
## Відповідність вимогам лабораторної роботи
 
| Вимога | Статус | Деталі |
|--------|--------|--------|
| Розпізнавання літер англійської абетки (верхній і нижній регістр) | ✅ | `AsciiState` через `symbol.isalpha()` |
| Розпізнавання цифр | ✅ | `AsciiState` через `symbol.isdigit()` |
| Оператор `*` (нуль або більше) | ✅ | `StarState` з бектрекінгом |
| Оператор `+` (один або більше) | ✅ | `PlusState` з бектрекінгом |
| Оператор `.` (будь-який символ) | ✅ | `DotState` завжди повертає `True` |
| Патерн State (ООП) | ✅ | Абстрактний клас `State`, всі стани наслідують його |
| Клас `RegexFSM` компілює вираз | ✅ | `__init__` будує ланцюжок станів |
| Перевірка рядків скомпільованим автоматом | ✅ | `check_string()` з рекурсивним бектрекінгом |
 
---
 
## Архітектура та пояснення імплементації
 
### Патерн State
 
Кожен токен у regex-виразі перетворюється на об'єкт-стан (`State`). Усі стани наслідують абстрактний клас `State` та реалізують метод `check_self(char)` — чи може поточний стан «прийняти» даний символ.
 
```
StartState → [AsciiState | DotState | StarState | PlusState] → ... → TerminationState
```
 
### Класи станів
 
- **`StartState`** — початковий стан, точка входу автомата. Містить список наступних станів.
- **`TerminationState`** — кінцевий стан (не використовується явно у `check_string`, але передбачений архітектурою).
- **`DotState`** — стан для символу `.`; `check_self` завжди повертає `True` (приймає будь-який символ).
- **`AsciiState`** — стан для конкретної ASCII-літери або цифри; зберігає символ і порівнює з ним вхідний.
- **`StarState`** — обгортка для квантифікатора `*`; делегує `check_self` до внутрішнього стану (попереднього токена). Приймає нуль або більше збігів.
- **`PlusState`** — аналог `StarState` для `+`; вимагає щонайменше одного збігу.
### Компіляція виразу (`RegexFSM.__init__`)
 
При ініціалізації клас ітерує по символах regex-рядка. Для кожного символа:
1. Створюється відповідний стан (`DotState`, `AsciiState`, `StarState`, або `PlusState`).
2. При зустрічі `*` або `+` попередній стан **видаляється** з ланцюжка і **обгортається** у `StarState`/`PlusState`.
3. Новий стан додається до `next_states` попереднього.
Результат — лінійний ланцюжок станів у `StartState.next_states`.
 
### Перевірка рядка (`check_string`)
 
Використовує рекурсивний бектрекінг:
- Для звичайних станів — перевіряє символ і рухається далі.
- Для `StarState` / `PlusState` — жадібно споживає символи, потім перебирає всі можливі зупинки у зворотньому порядку (від найдовшого збігу до найкоротшого), щоб знайти варіант, який призводить до повного збігу.

## Інструкції до запуску
 
### Вимоги
 
- Python 3.10 або новіший (використовується `match`-оператор)
### Запуск
 
```bash
python regex_fsm.py
```
 
### Запуск власного прикладу
 
```python
from regex_fsm import RegexFSM
 
pattern = RegexFSM("a+b*c")
print(pattern.check_string("aabc"))   # True
print(pattern.check_string("bc"))     # False — потрібна хоча б одна 'a'
```

## Приклади запусків
 
### Вбудований приклад (`__main__`)
 
Regex-вираз: `a*4.+hi`
 
Розшифровка:
- `a*` — нуль або більше символів `a`
- `4` — рівно один символ `4`
- `.` — будь-який один символ
- `+` — один або більше попереднього (тобто будь-якого символу)
- `h` — рівно `h`
- `i` — рівно `i`
```
$ python regex_fsm.py
True
True
False
```
 
| Рядок | Результат | Пояснення |
|-------|-----------|-----------|
| `"aaaaaa4uhi"` | `True` | `a*`→6×`a`, `4`→`4`, `.+`→`u`, `hi`→`hi` |
| `"4uhi"` | `True` | `a*`→0×`a`, `4`→`4`, `.+`→`u`, `hi`→`hi` |
| `"meow"` | `False` | Немає відповідності шаблону |
 
### Додаткові приклади
 
```python
# Тільки цифри
r1 = RegexFSM("1+0*")
print(r1.check_string("1100"))  # True
print(r1.check_string("0001"))  # False
 
# Будь-яке слово з трьох символів
r2 = RegexFSM("...")
print(r2.check_string("abc"))   # True
print(r2.check_string("ab"))    # False
 
# Літери з квантифікаторами
r3 = RegexFSM("A*B+")
print(r3.check_string("AAABBB"))  # True
print(r3.check_string("BBB"))     # True
print(r3.check_string("AAA"))     # False — потрібна хоча б одна 'B'
```
 
---
 
## Обмеження
 
- Не підтримуються символи окрім ASCII-літер, цифр, `.`, `*`, `+`
- Немає підтримки груп `()`, класів символів `[]`, операторів `?`, `{n,m}`, `^`, `$`
- Оператори `*` і `+` можуть застосовуватись лише до одного попереднього токена (не до групи)
